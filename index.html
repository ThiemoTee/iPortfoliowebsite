Monitoring: automated logistics process

At an existing customer (logistics service provider) the logistics process of one warehouse is fully automated by controlling automatically driven roller conveyors and mobile shelving. The first version of this project is already in use, in the next phase the intention is, in addition to some yet to be implemented optimizations, to give the warehouse staff more insight into the operational flow through a yet to be fully established monitoring..

Problem analyse

The lack of this monitoring ensures that it is not visible to a logistics employee where potential problems are located. As a result, there is a risk that the fully automated process will be stopped or aborted. The employee is therefore unable to make an appropriate correction. In addition to the lack of monitoring, there are some optimization problems. The first is that the SSCC (barcode) is not always received. This namely due to a network problem when the scanners do not connect. Besides this, the current application is made in .NET which had to be converted to Node Red since this is the new standard in the company. Finally there are some optimization problems which are: the conveyor sometimes makes false pairs and single pallets should not be put away in a wall location.

Wish and demands

The purpose of adding the monitoring is to make the logistics process of the automated warehouse more efficient. This in terms of 3 important issues that are currently already present in the warehouse. Namely: the conveyor, logging and visualization of the different aisles. We can call this project a success when we have reached the goal for the 3 points mentioned below.

Conveyor

During the process of the conveyor sometimes something can go wrong. Like the barcode of the pallet cannot be scanned In the current installation when something goes wrong a blue light just goes on, but unfortunately this doesn't give much information which makes that the employee knows that something is wrong but still has to look for help so that he/she can solve the problem. Therefore it would be interesting to have a dashboard where the employee can see all the necessary information so that when there is a problem there is a better chance that he can solve the problem himself. Finally, animating the conveyor would be a nice bonus since an image with animation can say much more than words.

Logging dashboard

At the moment there is a logging dashboard, but since it is not always that clear to read we want to clarify it. A nice addition would be a search bar to search for certain keywords in a file. By applying these elements it becomes more clear what would be a big help for the helpdesk.

Automatic racks

At the moment, due to a lack of clarity as to which aisle is open or should be open, palettes are set incorrectly. If we clarify this by creating a dashboard with the necessary information on it, the pallets will be put in the right place, which will avoid problems later on. A bonus is that there is also a floor plan available where you can see the exact status of all the shelves.
